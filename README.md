## Задание 2
Код реализован по принципу **Page Object**: логика работы со страницами и элементами вынесена в отдельные классы.

### Базовые классы

- **BasePage** — базовый класс страницы: хранит корневой элемент страницы и общие проверки/поведение, которые переиспользуются всеми объектами.
- **BaseElement** — базовый класс элемента: обёртка над `SelenideElement`, которая позволяет описывать переиспользуемые UI-элементы (например, ссылки, хедер) единообразно.

### Наследуемые классы

От базовых абстрактных классов наследуются конкретные страницы и элементы, необходимые для отдельных тестов (например, **MainPage**, **DocumentationPage**, **Header**, **Link**, а также утилиты вроде **TabSwitcher**).

## Реализованные тесты
### 1. Корректное отображения панели на главной странице `MainPageHeaderTest`
### 1.1. Отображение верхней панели (Header) на главной странице `mainPage_shouldShowHeader()`
**Цель:** Убедиться, что header отображается после открытия главной страницы.

**Шаги:**
1. Открыть главную страницу: https://selectel.ru/
2. Найти верхнюю панель (Header) на странице.
3. Проверить, что header отображается (visible/displayed).

**Ожидаемый результат:**
- Верхняя панель (Header) присутствует на странице и видима пользователю.

---

### 1.2. Отображение логотипа Selectel в верхней панели (Header) на главной странице `mainPage_shouldShowLogoInHeader()`
 
**Шаги:**
1. Открыть главную страницу: https://selectel.ru/
2. Найти верхнюю панель (Header).
3. Внутри header найти элемент логотипа Selectel.
4. Проверить, что логотип отображается (visible/displayed).

**Ожидаемый результат:**
- Логотип Selectel присутствует в header и видим пользователю.

---

### 2. Открытие раздела “Документация” в новой вкладке `OpenDocumentationTest`

**Шаги:**
1. Открыть главную страницу: https://selectel.ru/
2. В верхнем меню нажать на ссылку “Документация” (элемент `a[data-qa="main-nav__link-documentation"]`).
3. Переключиться на вновь открывшуюся вкладку.

**Ожидаемый результат:**
- Открыта новая вкладка браузера.
- URL активной вкладки начинается с `https://docs.selectel.ru/`.
- Страница документации отображается (страница загружена, `body` видим).

## Какие инструменты используются

- **Java 17** — язык и платформа выполнения.
- **Maven** — сборка проекта и запуск тестов (`mvn test`).
- **JUnit 5 (JUnit Jupiter)** — фреймворк для написания и запуска тестов.
- **Selenide** — высокоуровневая обёртка над Selenium для UI-тестов (локаторы, ожидания, действия).
- **Selenium / ChromeDriver** — управление браузером Google Chrome.

# Локальный запуск через Docker Compose

## Требования:
- Docker (Docker Desktop под Windows + WSL2 или Docker Engine под Linux)

Проверка:
```bash
docker context show
```
Должно быть `default`, eсли нет:
```bash
docker context use default
```

## Запуск тестов
```bash
# 1. Запустить Selenium Chrome в фоне
docker compose up -d chrome

# 2. Собрать образ и запустить тесты (при первом запуске или после изменений)
docker compose run --rm tests mvn test
```

### Посмотр логов:
```bash
docker compose logs --tail=200 tests
```

### Остановка и очистка:
```bash
docker compose down
```

### Проверка статуса контейнеров:
```bash
docker compose ps -a
```
